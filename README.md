# Сервис уведомлений о событиях календаря через VK

Тренировочный backend-сервис для автоматического мониторинга календаря пользователя, обработки событий по заданным правилам и отправки персонализированных уведомлений в личные сообщения ВКонтакте.

**Статус:** Основной функционал реализован, ведётся доработка тестов и документации.

## Архитектура-и-потоки данных

Сервис построен по модульной архитектуре с разделением на слои:

- **API** (FastAPI) – REST-интерфейс для клиентов и обработчик webhook от VK.
- **Бизнес-логика** (сервисы) – группировка, режим тишины, конфликты, команды откладывания.
- **Интеграции** – клиенты для Google Calendar, Google OAuth, VK API.
- **Планировщик** (APScheduler) – периодические задачи синхронизации, отправки напоминаний, еженедельных сводок.
- **Хранилище** – SQLite через SQLAlchemy, модели для пользователей, токенов, событий, истории уведомлений.

### Поток данных (упрощённый)
[Google Calendar] ←—периодическая синхронизация— [Сервис]  
↓  
[Локальная БД событий] → [Планировщик] → [VK API] → [Пользователь VK]  
↑  
[Пользователь] → [REST API] (настройки, история)  
[Пользователь] → [Callback VK] (команды откладывания) → [Сервис] → обновление напоминаний  


1. **Регистрация и подключение календаря**  
   Пользователь регистрируется с VK ID, затем авторизует сервис через Google OAuth2. Токены шифруются и сохраняются в БД.

2. **Синхронизация событий**  
   Раз в 5 минут планировщик обходит всех активных пользователей и загружает события из Google Calendar за ближайшие 7 дней. Новые и изменённые события сохраняются, удалённые помечаются как `cancelled`.

3. **Планирование уведомлений**  
   Для каждого активного события создаётся до трёх заданий в планировщике (по умолчанию за 60, 15, 5 минут). При изменении времени события задания пересоздаются.

4. **Отправка уведомлений**  
   В заданное время задание активируется. Перед отправкой применяются фильтры:
   - Режим тишины (если текущее время в запрещённом интервале и событие не исключение – уведомление отменяется).
   - Группировка (если несколько событий начинаются в пределах настроенного окна, формируется одно сводное сообщение, остальные задания удаляются).

5. **Еженедельная сводка**  
   Раз в неделю (настраиваемый день и время) генерируется сообщение с планом на 7 дней: количество событий, приоритетные встречи, свободные окна.

6. **Конфликты**  
   После каждой синхронизации проверяются пересечения событий. При обнаружении планируется предупреждение за 1 час до первого конфликта.

7. **Команды откладывания (snooze)**  
   Пользователь может ответить на сообщение бота командой (`+10`, `+1ч`, `завтра`, `отмена`). Входящие сообщения принимаются через Callback API VK, парсятся, соответствующие напоминания переносятся или отменяются.

## Текущий функционал

### Функционал
- Подключение к Google Calendar через REST API.
- OAuth-авторизация и безопасное хранение токенов (шифрование Fernet).
- Периодическая синхронизация событий (каждые 5 минут).
- Отправка уведомлений во ВКонтакте (messages.send).
- Базовые напоминания (три интервала, настраиваемые).
- Изоляция данных нескольких пользователей.
- REST API для истории событий, уведомлений и изменения настроек (частично).
- Логирование действий и ошибок.

### Бизнес-логика
- **Группировка уведомлений по временным окнам**  
  Настраиваемый интервал (по умолчанию 2 часа). При изменении/отмене одного события сводное сообщение корректно обновляется.

- **Режим ограничения уведомлений с исключениями**  
  Задаётся интервал тишины (например, 23:00–08:00). Исключения – по ключевым словам в названии события.

- **Еженедельная аналитическая сводка**  
  Автоматическая генерация каждую неделю (день и время настраиваются). При простое сервиса отправляется сразу после восстановления (однократный сдвиг).

- **Обработка команд откладывания**  
  Распознаются команды `+10`, `+1ч`, `завтра`, `отмена`. Ответное подтверждение отправляется пользователю.

- **Автоматическое обнаружение и уведомление о конфликтах**  
  Сравниваются интервалы событий. За 1 час до первого конфликта отправляется предупреждение с описанием пересекающихся событий.

### Дополнительно
- Журнал уведомлений и ошибок в БД.
- Повторные попытки отправки с экспоненциальной задержкой (до 3 попыток).
- Обработка часовых поясов (временная зона пользователя учитывается при расчётах).

## Локальный запуск

### Предварительные требования

- Python 3.10+
- pip
- Учётная запись Google Cloud с проектом и включённым Calendar API
- Группа ВКонтакте с настроенным Callback API (для команд откладывания) или токен группы для отправки сообщений

### Установка и настройка

1. Клонируйте репозиторий:  
    git clone https://github.com/Dementro1/VK_calendar  
    cd ..\VK_calendar

2. Создайте виртуальное окружение и активируйте его:  
    python -m venv venv  
    source venv/bin/activate  # Windows: venv\Scripts\activate

3. Установите зависимости:  
    pip install -r requirements.txt

4. Переименуйте файл .env.example в .env и заполните все переменные

5. Сгенерируйте ключ шифрования (если ещё нет) и запишите в .env переменную ENCRYPTION_KEY  
    python -c "from cryptography.fernet import Fernet; print(Fernet.generate_key().decode())"

6. Получите учетные данные Google Cloud:
    - Перейдите в Google Cloud Console.
    - Создайте проект, включите Google Calendar API.
    - В разделе "Учетные данные" создайте OAuth 2.0 Client ID типа "Веб-приложение".
    - Укажите разрешённый URI перенаправления: http://localhost:8000/auth/google/callback.
    - Скопируйте Client ID и Client Secret в .env как GOOGLE_CLIENT_ID и GOOGLE_CLIENT_SECRET.

7. Получите токен группы ВКонтакте:
    - Создайте группу или используйте существующую.
    - В настройках группы → "Работа с API" → "Ключи доступа" создайте токен с правом messages.
    - Запишите VK_GROUP_TOKEN и VK_GROUP_ID в .env.
    - Для приёма команд откладывания:
    - Включите Callback API в настройках группы.
    - Укажите  URL вашего сервера
    - Скопируйте код подтверждения в VK_CONFIRMATION_CODE.
    - Включите событие "Входящее сообщение".

### Инициализация базы данных
При первом запуске сервера все необходимые таблицы создаются автоматически (вызов Base.metadata.create_all в lifespan). Ручных действий не требуется.

### Запуск сервера
    uvicorn src.main:app --reload
Сервер будет доступен по адресу http://localhost:8000.  
Интерактивная документация API: http://localhost:8000/docs.

## REST API

### Аутентификация и регистрация
- POST /auth/register – регистрация пользователя по VK ID

Запрос:  

    {
        "vk_id": 123456789 
    }

- GET /auth/google/login?user_id=1 – получение URL авторизации Google  

Ответ:  

    {
        "user_id": 1,
        "message": "User registered successfully"
    }

### Настройки уведомлений
- GET /settings/?user_id=1 – получить настройки

Ответ:  

    {
        "user_id": 1,
        "reminder_intervals": [60, 15, 5],
        "silence_start": "23:00",
        "silence_end": "08:00",
        "silence_exceptions": ["врач", "срочно", "семья", "дедлайн"],
        "grouping_window": 120,
        "weekly_summary_day": 6,
        "weekly_summary_time": "09:00"
    }

- PUT /settings/?user_id=1 – обновить настройки  

Тело запроса:

    {
        "reminder_intervals": [30, 10],
        "silence_start": "22:00",
        "silence_end": "07:00",
        "silence_exceptions": ["срочно"],
        "grouping_window": 60,
        "weekly_summary_day": 0,
        "weekly_summary_time": "10:00"
    }

### Настройки уведомлений
GET /users/me – (заглушка) информация о текущем пользователе

Ответ:

    {
    "id": 1,
    "vk_id": 12345,
    "timezone": "Europe/Moscow"
    }

## Callback API VK
Эндпоинт для приёма событий от ВКонтакте.

POST /vk/callback – обрабатывает подтверждение сервера и новые сообщения.

Формат данных описан в документации VK.

При получении сообщения, если текст соответствует команде откладывания (+10, +1ч, завтра, отмена), сервис применяет команду к последнему отправленному уведомлению и отвечает подтверждением.
